# Supported Features

This is a non-exhaustive list of features that are supported by Pilot.js.

## Supported Next.js features

If a feature is not in the table below, that likely means there are no plans to support it or we may have missed it.  However, you can still use unsupported features in the web environment.

| Feature              | Supported           | Description |
|----------------------|---------------------|-------------|
| [Pages](https://nextjs.org/docs/basic-features/pages) | <center>‚úÖ</center> | Automatically finds routes from `/pages` or `/src/pages` directory via [`pilot build`](https://pilot.waveplay.dev/docs/cli.md) command. |
| [Custom App](https://nextjs.org/docs/advanced-features/custom-app) | <center>‚úÖ</center> | Optional app wrapper is applied on pages when the [`_app`](https://nextjs.org/docs/advanced-features/custom-app) route is registered. |
| [Custom Error Page](https://nextjs.org/docs/advanced-features/custom-error-page) | <center>‚úÖ</center> | You may provide custom error pages by registering `/404` and `/500` routes. |
| [getServerSideProps](https://nextjs.org/docs/basic-features/data-fetching/get-server-side-props) | <center>‚úÖ</center> | Data fetching uses the native runtime by default. See [Web Props](/docs/web-props) to use the Node server runtime instead. |
| [getStaticProps](https://nextjs.org/docs/basic-features/data-fetching/get-static-props)     | <center>‚úÖ</center> | Data fetching uses the native runtime by default. See [Web Props](/docs/web-props) to use the Node server runtime instead.<br/><br/>Native runtime caching is an opt-in feature. See [Caching](/docs/caching) for more information. |
| [Internationalization](https://nextjs.org/docs/advanced-features/i18n-routing) | <center>‚úÖ</center> | Internationalized routing is supported via the `i18n` config option. [Learn more](/docs/i18n). |
| [Env Variables](https://nextjs.org/docs/basic-features/environment-variables)	| <center>‚úÖ</center> | Environment variables are loaded during `pilot build`. This follows the [same pattern](https://nextjs.org/docs/basic-features/environment-variables#exposing-environment-variables-to-the-browser) and [load order](https://nextjs.org/docs/basic-features/environment-variables#environment-variable-load-order) as Next.js. |
| [Redirects](https://nextjs.org/docs/api-reference/next.config.js/redirects) | <center>‚úÖ</center> | Redirects are supported by [returning a `redirect` object](https://nextjs.org/docs/api-reference/data-fetching/get-server-side-props#redirect) from your props functions or via a `redirects` function in your config file(s). [Learn more](/docs/redirects). |
| [Context Parameter](https://nextjs.org/docs/api-reference/data-fetching/get-server-side-props#context-parameter) | <center>üèóÔ∏è</center> | `context` is passed to `getServerSideProps` and `getStaticProps` functions. | [next/link](https://nextjs.org/docs/api-reference/next/link) | <center>üèóÔ∏è</center> | Supports most functionality that `next/link` provides via `@waveplay/pilot/link` export.<br/><br/>Props not supported yet:<br/>`passHref`, `prefetch`, `replace`, `scroll`, `shallow` |
| [next/router](https://nextjs.org/docs/api-reference/next/router#userouter) | <center>üèóÔ∏è</center> | Supports most functionality using the `usePilot()` hook or the `withRouter` HOC.<br/><br/>Fields not supported yet:<br/>`basePath`, `domainLocales`, `isFallback`, `isReady`, `isPreview`.<br/><br/>Functions not supported yet:<br/>`beforePopState()`, `events()`, `prefetch()`, `replace()` |
| [Rewrites](https://nextjs.org/docs/api-reference/next.config.js/rewrites) | <center>üèóÔ∏è</center> | Supported via `rewrites` field in config file.<br/><br/>`beforeFiles`, `afterFiles`, and `fallback` are not supported yet. They are currently treated the same. |
| [getStaticPaths](https://nextjs.org/docs/basic-features/data-fetching/get-static-paths) | <center>‚ùå</center> | Not supported yet. |
| [API Routes](https://nextjs.org/docs/api-routes/introduction) | <center>‚ùå</center> | Not supported yet. |
| [Middleware](https://nextjs.org/docs/advanced-features/middleware) | <center>‚ùå</center> | Not supported yet. |
| [Static Export](https://nextjs.org/docs/advanced-features/static-html-export) | <center>‚ùå</center> | Not supported yet. |
| [App Directory](https://beta.nextjs.org/docs/upgrade-guide#migrating-from-pages-to-app) | <center>‚ùå</center> | Not supported yet. |
| [Server Components](https://nextjs.org/docs/advanced-features/react-18/server-components) | <center>‚ùå</center> | Not supported yet. |

> **Good to know:** Native apps work much differently than browsers and servers, meaning some features may not be possible to support. Please open an issue if you'd like to see a feature added.
